# 拓扑排序

## 1. 核心思想
用于有向无环图中的依赖顺序。Kahn 算法不断处理入度为 0 的节点。

## 2. 适用题型
- 课程先修关系
- 任务依赖、构建顺序
- 判断有向图是否存在环
- 返回一种合法执行顺序

## 3. 经典例题和逐步拆解
LeetCode 207：课程表。

1. 按 `prerequisite -> course` 建邻接表。
2. 统计每门课程入度。
3. 所有入度为 0 的课程入队。
4. 完成一门课程后，后续课程入度减一。
5. 新入度为 0 的课程继续入队。
6. 最终完成数量等于课程数，说明无环。

复杂度：时间 `O(V + E)`，空间 `O(V + E)`。

## 4. 易错点
- 把边方向写反。
- 减少当前节点自己的入度，而不是后续节点。
- 使用 `<= 0` 导致节点重复入队。
- 用“队列是否为空”判断是否有环；应比较处理节点总数。

## 5. 今日练习题
LeetCode 210：课程表 II。在 BFS 出队时收集课程顺序；若结果长度不足则返回空数组。

## 6. 精华总结
有向依赖 + 合法顺序，优先想拓扑排序；处理不完所有节点意味着存在环。
