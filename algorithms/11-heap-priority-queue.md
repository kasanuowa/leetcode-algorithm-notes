# 堆 / 优先队列

## 1. 核心思想
堆用于动态维护最大值或最小值。小顶堆堆顶最小，大顶堆堆顶最大。

常见复杂度：`push/pop = O(log n)`，`peek = O(1)`。

## 2. 适用题型
- 第 K 大 / 第 K 小
- Top K 高频元素
- 合并多个有序序列
- 数据流中的中位数、任务调度

## 3. 经典例题和逐步拆解
LeetCode 215：数组中的第 K 大元素。

1. 维护大小为 `k` 的小顶堆。
2. 每个数字入堆。
3. 若堆大小超过 `k`，弹出最小值。
4. 最后堆中保留最大的 `k` 个数，堆顶即第 `k` 大。

复杂度：时间 `O(n log k)`，空间 `O(k)`。

## 4. 易错点
- 第 K 大使用大小为 K 的小顶堆，容易直觉写反。
- 忘记限制堆大小。
- 误以为堆内部完全有序；堆只保证堆顶。
- `bubbleDown` 时必须同时比较左右孩子。

## 5. 今日练习题
LeetCode 347：前 K 个高频元素。先用 `Map` 统计频率，再按频率维护小顶堆。

## 6. 精华总结
要完整排序用 `sort`；只关心前 K 个或不断取极值，优先考虑堆。
