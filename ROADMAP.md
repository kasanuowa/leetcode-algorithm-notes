## 当前规则

每天 10:30 学习一个 LeetCode 算法专题。

每节专题课固定分为 6 段：

```txt
1. 核心思想
2. 适用题型
3. 经典例题和逐步拆解
4. 易错点
5. 今日练习题
6. 精华总结
```

复习机制：

```txt
每上 5 天专题课，第 6 天改为综合练习课。
综合练习课不讲新专题，题目总数控制在 8 道左右。
```

## 当前掌握的题型判断树

```txt
要快速查“之前有没有”？ -> 哈希表
数组有序 / 左右收缩 / 原地整理？ -> 双指针
连续子数组 / 连续子串 / 最长最短？ -> 滑动窗口
有序数组查位置 / 第一个满足条件？ -> 二分
最近未匹配 / 后进先出？ -> 栈
一层一层扩散 / 最短步数？ -> 队列 / BFS
修改 next 指向 / 节点连接关系？ -> 链表
```

## 复盘方式

每道题写完后，只问 4 个问题：

```txt
1. 我为什么判断它属于这个专题？
2. 这题的核心变量代表什么？
3. 我什么时候更新答案 / 修改指针 / 入队出队？
4. 最容易错的边界是什么？
```

不要追求“看一遍就会”。算法学习更像健身：

```txt
第一次认识动作
第二次能模仿
第三次开始稳定
第四次才会真的长进脑子里
```

脑子觉得满，不是失败，是缓存快爆了。这个时候要做的是复盘和归类，不是继续猛灌。
